This documentation uses doxygen created from lua code. As some of you probably know doxygen doesn't support lua on it's own so some of the shown descriptions arent perfectly correct.

On this page some of the caveats are explained:
Lua doesn't support classes and structs but tables which ain't supported by doxygen.

Mapping of classes and structs:

  • classes are used to group (sub)components containing primary functions
  • structs are result of parsing tables and containing configuration or temporary data

Datatypes shown in doxygen and it's meaning:

  • function this is a return value of function and can be nil too
  • var a variable
  • struct some table element
  • anonymous_value a table element specified without a name
  • parameter is a parameter declared in a function definition

Keywords used in config parameter description

  • MANDATORY some parameter required
  • OPTIONAL parameter may be nill without causinh an error
  • MOV_GEN_DEPENDENT parameter is required dependent of selected movement gen
  • 2D MANDATORY parameter is required in case of 2D mob
  • 3D MANDATORY parameter is required in case of 3D mob
  • ALGORITHM DEPENDENT is required dependent on selected algorithm
